# Canonical Screen Audit

Last reviewed: `2026-03-24`

This file supersedes `docs/flow.md`, `docs/task.md`, and `docs/studio.md` for screen-level planning. Those files remain in the repo as historical reference and are not deleted.

Coverage source:

- Audited the current `src/app/**/page.tsx` tree as the primary source of truth.
- Covered all `100` page routes exactly once.
- Used older docs only as secondary input, then reconciled them against the current codebase.

Status scale:

- `Complete`
- `Mostly Complete`
- `Partial`
- `Placeholder`
- `Redirect`
- `Broken`

Completion values are pragmatic heuristics for how complete the real user journey feels today, not a strict implementation metric.

## System Gaps

### 1. Route and navigation drift

- `PATH.SITE.SUPPORT.CONTACT` still points to `/support/contact-support`, but the real page is `/support/contact`.
- `PATH.STORE.PRODUCTS.*` still points to `/store/products/*`, but the current storefront uses `/store/[categorySlug]/[subCategorySlug]/[variantSlug]`.
- `PATH.STUDIO.MARKETING.*` still advertises `campaigns`, `newsletters`, `promotions`, `coupons`, and `email-templates` routes that do not exist yet.
- The footer still links to dead destinations such as `/newsletter`, `/orders/track`, `/support/size-guide`, `/store/new`, `/store/best-sellers`, `/store/sale`, and `/store/gift-cards`.
- Shipping config table empty states still point to `/studio/shipping?view=providers|methods|zones|rates` even though dedicated create pages now exist.
- The account profile email-change callback still uses `/profile`, which has no route behind it.

### 2. Pricing, totals, and money display are still inconsistent

- `src/module/cart/components/cart-summary.tsx` still hardcodes free shipping and estimated tax.
- `src/module/checkout/components/checkout-summary.tsx` still hardcodes shipping, tax, and discount values instead of showing authoritative order pricing.
- `src/module/product/product-pdp.tsx` still displays PDP pricing in dollars while the broader app uses INR-style flows elsewhere.
- These gaps directly affect `/account/cart`, `/store/checkout`, and `/store/[categorySlug]/[subCategorySlug]/[variantSlug]`.

### 3. Commerce state sequencing still needs hardening

- Existing business logic audits still apply: order finalization, inventory deduction, and discount usage happen too early relative to confirmed payment.
- This is mostly a backend/system issue, but it is surfaced through checkout, payment status, and stock visibility screens.

### 4. Access control and ownership rules are not fully aligned

- Some studio routes allow any non-customer user at the page level, while the underlying API is admin-only.
- `payment.getStatus` still lacks an ownership check on `orderId`, which affects account payment history safety.
- Some customer-owned resource checks are still inconsistent across order and shipment APIs for non-admin roles.

### 5. Inventory and warehouse support are real but still uneven

- Inventory create/edit flows now exist, but they still rely on raw `variantId` and `warehouseId` inputs instead of safer pickers.
- Inventory detail/edit routes still need explicit `notFound()` handling.
- The warehouse-aware schema is ahead of the cart, checkout, and fulfillment runtime, so stock flows are not fully coherent yet.

### 6. Public support and content flows are still mostly skeletal

- The support tree is route-complete but workflow-incomplete.
- `About`, `Newsletter`, and the legal root all need real destination content instead of mostly shell-level composition.

### 7. Partial admin workflows remain

- The studio dashboard is still mostly placeholder metrics.
- Studio order detail still shows inert invoice/action buttons.
- Studio review moderation has a backend mutation but no real moderation UI.
- Studio discount edit needs proper preloaded data before it can feel complete.
- Studio users still parses `banned` and `emailVerified` filters without sending them into the actual query.

### 8. Verification note

- `pnpm run check` is not a clean acceptance gate for this documentation task right now because the repo already has unrelated formatting issues in `package.json`, `migrations/meta/*.json`, and some newer page files.
- That existing formatter noise should be cleaned up separately from this screen audit.
